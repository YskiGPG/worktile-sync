## 工作流程要求

每次完成代码修改后，你必须自动执行以下步骤，无需我提醒：

1. **自查**：重新阅读你刚写/改的代码，检查逻辑错误、边界情况、类型问题
2. **运行验证**：执行相关的 lint、typecheck、测试命令确认没有报错
3. **风险评估**：简要列出这次改动可能引入的风险或遗漏
4. **完整性检查**：确认改动是否完整——有没有漏改的文件、缺少的 import、未处理的错误路径

如果发现问题，直接修复，不要等我指出。

## 代码质量底线

- 不要留下 TODO 或占位代码，除非我明确要求
- 每个函数要考虑错误处理
- 改动涉及 API 的，检查请求和响应的类型是否匹配

完成后，以资深工程师的标准审查你自己的代码，发现问题直接修复，不需要问我。反复审查直到你确信没有问题。
